# Changelog

## 2026-06-05

### 架构观察
- 当前站点采用纯静态 HTML + 单一全局 CSS 的轻量结构，部署简单，但公共区块需要手动维护。
- 首页、关于页、文章页和 404 页原先存在结构不统一的问题，后续继续加页面时容易出现样式和导航分叉。
- 内容组织已经有了清晰方向，适合继续保持静态站点路线，再逐步补模板化和内容规范。

### 本次优化
- 统一了首页、关于页、文章页和 404 页的页面骨架、导航和页脚风格。
- 为主要页面补充了 `viewport`、`description` 和 `theme-color` 等基础元信息。
- 修正了 `about.html` 的错误返回路径，以及 `404.html` 缺失的结构闭合问题。
- 修正了文章页中列表嵌套在段落里的无效 HTML 结构。
- 重写了 `style.css`，加入颜色变量、卡片层级、响应式布局、焦点态和更清晰的内容分区。
- 补充了首页的学习进度区和更明确的文章入口文案。

### 后续建议
- 如果页面继续增多，建议引入简单模板流程，减少导航和页脚的重复维护。
- 可以继续增加文章元信息，例如发布时间、标签和更新状态。
- 若后续准备系统化写文章，建议把文章列表改为统一的数据清单驱动。
